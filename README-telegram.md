# راهنمای دریافت خودکار نرخ طلا از کانال @etjmir (بدون نیاز به ربات)

## خبر خوب
کانال شما (`https://t.me/etjmir` — اتحادیه صنف طلا، جواهر، نقره و سکه مشهد
مقدس) یک کانال **عمومی** است. یعنی نیازی به ساختن ربات تلگرام یا گرفتن
اجازه از مدیر کانال نیست؛ سایت‌ها می‌توانند مستقیماً صفحه‌ی وب این کانال
(`https://t.me/s/etjmir`) را بخوانند، همان‌طور که هرکسی بدون داشتن تلگرام
هم می‌تواند در مرورگر پیام‌های این کانال را ببیند.

پیام‌های نرخ در این کانال همیشه به همین شکل هستند:
```
گرم‌طلای18عیار : 17,250,000 تومان
```
فایل آماده‌شده (`fetch_rate.php`) دقیقاً همین الگو را از آخرین
پیام‌های کانال استخراج می‌کند.

## مراحل راه‌اندازی (فقط وقتی سایت روی هاست واقعی باشد)

۱. فایل `fetch_rate.php` را (همان‌طور که هست، نیازی به تغییر ندارد)
   روی هاست سایت آپلود کنید.

۲. از پنل هاست خود (مثلاً سی‌پنل) یک **Cron Job** بسازید که هر ۳ تا ۵
   دقیقه این دستور را اجرا کند:
   ```
   */5 * * * * php /home/username/public_html/fetch_rate.php
   ```

۳. این اسکریپت هر بار فایل `rate.json` را با آخرین نرخ به‌روز
   می‌کند، مثلاً:
   ```json
   {"rate": 17250000, "sourceText": "...", "updatedAt": "2026-07-01T12:00:00+00:00"}
   ```

۴. خود سایت (تابع `trySyncRateFromServer` در `ui.js`) در هر
   بار باز شدن هر صفحه، این فایل را می‌خواند و اگر نرخ جدیدتری پیدا کند،
   خودکار در تنظیمات فروشگاه ثبتش می‌کند — بدون هیچ دخالت دستی ادمین.

   همچنین از پنل ادمین → «نرخ طلا و تلگرام» → دکمه‌ی «دریافت نرخ الان»
   هم می‌توانید هر وقت خواستید این هماهنگ‌سازی را دستی هم اجرا کنید.

## چرا الان (روی نسخه‌ی تستی) کار نمی‌کند؟
چون فایل PHP فقط روی یک **سرور واقعی** اجرا می‌شود (نه در مرورگر شما).
همین الان که دارید سایت را از localStorage تست می‌کنید، این فایل هنوز
جایی اجرا نمی‌شود. به محض این‌که سایت را (طبق صحبت قبلی) به یک هاست واقعی
وصل کردیم، همین امروز این بخش را هم فعال می‌کنم — کار اضافه‌ای از طرف شما
لازم نیست، چون کانال از قبل عمومی است.

## اگر روزی کانال یا فرمت پیام‌ها عوض شد
اگر یوزرنیم کانال عوض شد، فقط مقدار `$CHANNEL_USERNAME` در بالای فایل
`fetch_rate.php` را عوض کنید. اگر فرمت متن پیام‌ها تغییر کرد (مثلاً
عیار دیگری اعلام شد)، مقدار `$RATE_REGEX` باید متناسب با متن جدید به‌روز
شود — این کار را هروقت لازم شد برایتان انجام می‌دهم.
